# BrewLog 프론트엔드 화면 설계서 v1.0

## 1. 문서 개요

이 문서는 브루잉 커피 기록 웹 서비스 **BrewLog**의 프론트엔드 화면 구조와 사용자 이동 흐름을 정리한 문서이다.

본 문서는 예시 이미지나 시각 자료 없이, 실제 구현에 필요한 다음 내용을 중심으로 작성한다.

- 전체 화면 구조
- 페이지별 역할
- 주요 UI 구성 요소
- 버튼/카드/메뉴 클릭 시 이동 흐름
- 사용자 입장에서의 화면 사용 흐름
- 심플하고 간결한 디자인 방향
- 메인 페이지 중심 설계

---

## 2. 프론트엔드 설계 방향

### 2.1 핵심 방향

BrewLog는 사용자가 많은 정보를 한 번에 소비하는 서비스가 아니라, 자신이 마신 커피 경험을 조용히 기록하고 다시 돌아보는 서비스이다.

따라서 프론트엔드의 핵심 방향은 다음과 같다.

```text
간결함
기록 중심
지도 중심
카드 기반
적은 정보량
명확한 이동 흐름
```

### 2.2 디자인 원칙

#### 1. 메인 페이지는 세계 지도가 중심이다

사용자가 사이트에 들어왔을 때 가장 먼저 보는 화면은 통계 숫자나 긴 목록이 아니라, **지금까지 마신 원두의 원산지 세계 지도**여야 한다.

메인 화면의 역할은 다음과 같다.

- 내가 어떤 나라의 원두를 마셔왔는지 한눈에 보여준다.
- 커피 기록을 시각적으로 쌓아가는 재미를 준다.
- 원두 기록 서비스의 정체성을 명확히 보여준다.

#### 2. 한 화면에 너무 많은 정보를 보여주지 않는다

각 화면에서는 핵심 정보만 보여준다.

예를 들어 원두 카드에는 모든 정보를 넣지 않는다.

좋은 카드 예시:

```text
Ethiopia Chelbesa
Fritz Coffee
Washed · Ethiopia
평점 4.5
```

나쁜 카드 예시:

```text
Ethiopia Chelbesa Washed, Fritz Coffee, Ethiopia, Yirgacheffe, Gedeb, Chelbesa,
Heirloom, 1,950m, Light Roast, 2026-04-28, 200g, 18,000원,
Jasmine, Peach, Lemon Tea, 산미 좋음, ...
```

상세 정보는 상세 페이지에서 확인하게 한다.

#### 3. 사용자는 항상 다음 행동을 쉽게 찾을 수 있어야 한다

각 페이지에는 사용자가 할 수 있는 대표 행동이 명확해야 한다.

예:

- 메인 페이지: 원두 등록하기, 기록 남기기
- 원두 목록: 원두 추가하기
- 원두 상세: 브루잉 기록 남기기
- 브루잉 기록 상세: 수정하기
- 카페 지도: 방문 기록 추가하기

#### 4. 리스트와 상세를 분리한다

목록 페이지는 탐색용이다.  
상세 페이지는 읽기용이다.  
등록/수정 페이지는 입력용이다.

각 페이지의 목적이 섞이지 않게 한다.

## 3. 전체 메뉴 구조

### 3.1 상단 내비게이션 메뉴

상단 메뉴는 너무 많지 않게 구성한다.

```text
BrewLog | 홈 | 원두 | 브루잉 | 카페 | 통계
```

### 3.2 메뉴별 이동

| 메뉴 | 이동 경로 | 설명 |
|---|---|---|
| BrewLog 로고 | `/` | 메인 페이지로 이동 |
| 홈 | `/` | 메인 페이지로 이동 |
| 원두 | `/beans` | 원두 목록 페이지로 이동 |
| 브루잉 | `/brew-records` | 브루잉 기록 목록 페이지로 이동 |
| 카페 | `/cafe-visits` | 카페 방문 목록 또는 카페 지도 페이지로 이동 |
| 통계 | `/dashboard` | 통계 대시보드로 이동 |

### 3.3 추천 메뉴 정책

카페 메뉴를 클릭했을 때는 목록보다 **카페 지도 페이지**로 이동하는 것을 추천한다.

이유:

- 카페 기록은 위치 기반 경험과 잘 어울린다.
- 사용자가 방문한 카페를 지도에서 보는 재미가 크다.
- 목록은 지도 옆 보조 패널로 제공할 수 있다.

추천 구조:

```text
원두 클릭 → 원두 목록
브루잉 클릭 → 브루잉 기록 목록
카페 클릭 → 카페 지도
통계 클릭 → 통계 대시보드
```

# 5. 공통 레이아웃 설계

## 5.1 기본 레이아웃

모든 화면은 다음 구조를 기본으로 한다.

```text
[Header]
  로고 / 메뉴 / 등록 버튼

[Main]
  페이지별 콘텐츠

[Footer]
  간단한 저작권 또는 서비스명
```

## 5.2 Header 구성

### 좌측

```text
BrewLog
```

클릭 시 `/`로 이동한다.

### 중앙 또는 우측 메뉴

```text
홈
원두
브루잉
카페
통계
```

### 우측 주요 버튼

```text
+ 기록하기
```

`+ 기록하기` 버튼 클릭 시 작은 드롭다운을 보여준다.

드롭다운 항목:

```text
원두 등록
브루잉 기록
카페 방문
```

클릭 흐름:

| 클릭 항목 | 이동 |
|---|---|
| 원두 등록 | `/beans/new` |
| 브루잉 기록 | `/brew-records/new` |
| 카페 방문 | `/cafe-visits/new` |

## 5.3 화면 폭 정책

### 데스크톱

- 최대 콘텐츠 폭: 1100px ~ 1200px
- 중앙 정렬
- 지도 화면은 조금 더 넓게 사용 가능

### 모바일

- 상단 메뉴는 햄버거 메뉴로 전환 가능
- 지도 아래에 리스트 표시
- 입력 폼은 1열 구성

# 6. 메인 페이지 설계

## 6.1 URL

```text
/
```

## 6.2 페이지 목적

메인 페이지는 사용자가 지금까지 기록한 원두의 원산지를 세계 지도에서 한눈에 확인하는 화면이다.

이 서비스의 첫인상은 **기록 개수**가 아니라 **커피 원산지 지도**가 되어야 한다.

## 6.3 메인 페이지 핵심 구성

```text
[짧은 인사 문구]

[세계 지도 영역]

[선택된 국가 요약 카드]

[최근 기록 3개]

[빠른 액션 버튼]
```

## 6.4 화면 구성 상세

### 1. 상단 인사 영역

너무 크게 만들지 않는다.

예시 문구:

```text
나의 커피 여정
지금까지 마신 원두의 원산지를 모아봤어요.
```

또는:

```text
오늘도 한 잔의 기록을 남겨보세요.
```

표시 정보는 간단히 3개 정도만 둔다.

```text
원두 12개 · 브루잉 34회 · 방문 카페 5곳
```

### 2. 세계 지도 영역

메인 화면의 가장 큰 영역이다.

표시 내용:

- 원두 생산 국가에 마커 표시
- 국가별 원두 개수 표시
- 마커 클릭 시 국가 요약 정보 표시

지도 마커 예시:

```text
Ethiopia 5
Colombia 3
Kenya 2
Panama 1
```

### 3. 국가 마커 클릭 시 동작

사용자가 세계 지도에서 `Ethiopia` 마커를 클릭하면, 지도 옆 또는 아래에 요약 카드가 표시된다.

요약 카드:

```text
Ethiopia

기록한 원두 5개
평균 평점 4.3
대표 노트: Floral, Citrus, Peach

[이 국가 원두 보기]
```

버튼 클릭 흐름:

| 클릭 | 이동 |
|---|---|
| 이 국가 원두 보기 | `/beans?country=Ethiopia` |

### 4. 지도 빈 상태

아직 원두가 등록되지 않은 경우:

```text
아직 지도에 표시할 원두가 없어요.
첫 원두를 등록해보세요.
```

버튼:

```text
[원두 등록하기]
```

클릭 시:

```text
/beans/new
```

### 5. 최근 기록 영역

지도 아래에는 최근 기록을 너무 많이 보여주지 않는다.

추천:

```text
최근 브루잉 기록 3개
```

각 카드에는 핵심만 표시한다.

```text
Ethiopia Chelbesa
V60 · 15g / 250ml
평점 4.5
2026.05.02
```

카드 클릭 시:

```text
/brew-records/{recordId}
```

### 6. 빠른 액션 버튼

메인 페이지 하단 또는 우측에 다음 버튼을 둔다.

```text
[원두 등록]
[브루잉 기록]
[카페 방문]
```

클릭 흐름:

| 버튼 | 이동 |
|---|---|
| 원두 등록 | `/beans/new` |
| 브루잉 기록 | `/brew-records/new` |
| 카페 방문 | `/cafe-visits/new` |

# 7. 원두 목록 페이지 설계

## 7.1 URL

```text
/beans
```

## 7.2 페이지 목적

사용자가 등록한 원두를 간결한 카드 목록으로 확인하는 화면이다.

## 7.3 화면 구성

```text
[페이지 제목 + 원두 등록 버튼]

[검색 / 필터 / 정렬]

[원두 카드 목록]
```

## 7.4 상단 영역

제목:

```text
원두
```

보조 문구:

```text
기록한 원두를 한눈에 확인해보세요.
```

우측 버튼:

```text
+ 원두 등록
```

클릭 시:

```text
/beans/new
```

## 7.5 검색 / 필터 영역

너무 많은 필터를 한 번에 보여주지 않는다.

기본 노출:

```text
검색창
정렬 드롭다운
```

검색창 placeholder:

```text
원두 이름, 로스터리 검색
```

정렬 옵션:

```text
최근 등록순
이름순
평점 높은 순
구매일순
```

고급 필터는 접기/펼치기 방식으로 둔다.

```text
[필터]
```

필터 클릭 시 표시:

```text
국가
가공 방식
로스터리
```

## 7.6 원두 카드 구성

카드는 간결하게 구성한다.

표시 정보:

```text
원두 이름
로스터리
국가 · 가공 방식
대표 향미 노트 2~3개
평균 평점
```

예시:

```text
Ethiopia Chelbesa
Fritz Coffee
Ethiopia · Washed
Jasmine · Peach · Lemon
★ 4.5
```

## 7.7 카드 클릭 흐름

| 클릭 대상 | 이동 |
|---|---|
| 원두 카드 전체 | `/beans/{beanId}` |
| 로스터리명 | MVP에서는 별도 이동 없음 |
| 국가명 | `/beans?country={country}` |
| 향미 노트 | MVP에서는 별도 이동 없음 |

## 7.8 빈 상태

등록된 원두가 없는 경우:

```text
아직 등록한 원두가 없어요.
구매한 원두를 먼저 기록해보세요.
```

버튼:

```text
[첫 원두 등록하기]
```

클릭 시:

```text
/beans/new
```

# 8. 원두 등록 페이지 설계

## 8.1 URL

```text
/beans/new
```

## 8.2 페이지 목적

새 원두 정보를 등록하는 화면이다.

사용자는 직접 입력하거나, 원두 카드 이미지를 업로드하여 OCR로 정보를 자동 추출할 수 있다.

## 8.3 화면 구성

```text
[페이지 제목]

[원두 카드 이미지로 자동 입력 영역]

[기본 정보 입력]

[상세 정보 입력]

[구매 정보 입력]

[메모 입력]

[저장 / 취소 버튼]
```

## 8.4 상단 제목

```text
원두 등록
새로 구매한 원두를 기록해보세요.
```

## 8.5 OCR 자동 입력 영역

페이지 상단에 작게 제공한다.

```text
원두 카드가 있다면 사진으로 정보를 채울 수 있어요.

[이미지 업로드]
```

### 이미지 업로드 클릭 흐름

1. 사용자가 `[이미지 업로드]` 클릭
2. 파일 선택 창 표시
3. 이미지 선택
4. OCR 처리
5. 추출된 텍스트 표시
6. 자동 입력 후보 표시
7. 사용자가 확인
8. 입력 폼에 반영

### OCR 결과 확인 UI

```text
추출된 정보가 맞는지 확인해주세요.

원두 이름: Ethiopia Chelbesa
국가: Ethiopia
가공 방식: Washed
향미 노트: Jasmine, Peach, Lemon

[폼에 적용하기]
[다시 업로드]
```

`폼에 적용하기` 클릭 시:

- 해당 값들이 입력 폼에 채워진다.
- 사용자는 직접 수정 가능하다.

## 8.6 입력 폼 구성

한 번에 너무 많은 입력란이 보이면 부담스럽기 때문에 섹션을 나눈다.

### 기본 정보

항상 펼쳐져 있다.

```text
원두 이름 *
로스터리
국가
가공 방식
향미 노트
```

### 상세 정보

# 12. 브루잉 기록 등록 페이지 설계

## 12.1 URL

```text
/brew-records/new
```

또는 특정 원두에서 진입한 경우:

```text
/brew-records/new?beanId={beanId}
```

## 12.2 페이지 목적

직접 내린 커피의 추출 레시피와 맛 평가를 기록하는 화면이다.

## 12.3 화면 구성

```text
[페이지 제목]

[원두 선택]

[레시피 입력]

[맛 평가]

[메모]

[저장 / 취소]
```

## 12.4 원두 선택

### 일반 진입

`/brew-records/new`로 들어온 경우:

```text
원두 선택 *
```

드롭다운 또는 검색 선택 방식으로 원두를 선택한다.

### 원두 상세에서 진입

`/brew-records/new?beanId={beanId}`로 들어온 경우:

- 해당 원두가 미리 선택되어 있다.
- 사용자는 필요하면 다른 원두로 변경할 수 있다.

## 12.5 레시피 입력

기본적으로 자주 쓰는 항목만 보여준다.

```text
추출 날짜 *
추출 도구 *
원두량
물량
물 온도
분쇄도
추출 시간
```

### 추출 시간 입력 방식

사용자에게는 다음처럼 보이게 할 수 있다.

```text
분
초
```

저장할 때는 초 단위로 변환한다.

예:

```text
2분 30초 → 150
```

## 12.6 맛 평가

처음부터 너무 많은 평가 항목을 강제하지 않는다.

기본 표시:

```text
전체 평점
한줄 감상
```

상세 평가 접기/펼치기:

```text
[상세 맛 평가]
```

펼치면:

```text
산미
단맛
쓴맛
바디감
향
밸런스
```

## 12.7 저장 버튼 흐름

| 버튼 | 동작 |
|---|---|
| 저장 | 저장 후 `/brew-records/{recordId}` 이동 |
| 취소 | 이전 페이지 또는 `/brew-records` 이동 |

# 24. 초기 버전에서 제외해도 되는 화면

MVP 구현 부담을 줄이기 위해 아래는 나중으로 미뤄도 된다.

| 화면 | 제외 이유 |
|---|---|
| 로스터리 목록 | MVP에서는 문자열로 관리해도 충분함 |
| 원두 구매 이력 목록 | Bean에 구매 정보를 포함해도 충분함 |
| 향미 노트별 검색 화면 | 초기에는 텍스트 검색으로 충분함 |
| OCR 전용 페이지 | 원두 등록 페이지 내부 기능으로 처리 가능 |
| 캘린더 화면 | 기록이 쌓인 뒤 구현해도 됨 |
| 공개 프로필 페이지 | 개인 기록 서비스가 우선임 |

# 25. 최종 프론트 설계 요약

BrewLog의 프론트엔드는 **간결한 기록 서비스**를 목표로 한다.

가장 중요한 설계 결정은 다음과 같다.

1. **메인 페이지는 세계 지도 중심이다.**
   - 사용자가 지금까지 마신 원두의 원산지가 바로 보인다.
   - 지도는 서비스의 첫인상이 된다.

2. **목록 페이지는 카드 중심으로 간결하게 구성한다.**
   - 카드에는 핵심 정보만 보여준다.
   - 상세 정보는 상세 페이지에서 확인한다.

3. **등록 화면은 섹션을 나누어 입력 부담을 줄인다.**
   - 기본 정보만 먼저 보여준다.
   - 상세 정보는 접기/펼치기로 처리한다.

4. **카페 기록은 지도 중심으로 탐색한다.**
   - 카페 메뉴 클릭 시 카페 지도 페이지로 이동하는 것을 추천한다.

5. **상단의 `+ 기록하기` 버튼으로 빠르게 기록할 수 있게 한다.**
   - 원두 등록, 브루잉 기록, 카페 방문 기록으로 바로 이동한다.

초기 구현에서는 다음 페이지를 우선 개발하는 것이 적절하다.

```text
1. 홈: 원두 세계 지도
2. 원두 목록 / 등록 / 상세
3. 브루잉 기록 목록 / 등록 / 상세
4. 카페 지도 / 카페 방문 등록 / 상세
5. 통계 대시보드
```
